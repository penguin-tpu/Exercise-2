## Current Progress

- Extended the scratchpad timing model with explicit read-port and write-port reservations on top of the existing banked contention model.
- Added regressions that distinguish bank conflicts from port conflicts across DMA and load/store traffic.
- Revalidated the entire test suite after the new shared-resource wiring.

## Current Implementation Status

- `perf_modeling/isa/semantics.py` now assigns one scratchpad read or write port reservation per scratchpad access:
  - scalar loads use read ports
  - scalar stores use write ports
  - tensor loads use read ports
  - tensor stores use write ports
  - DMA uses source read ports and destination write ports when scratchpad is involved
- `perf_modeling/engine.py` now enforces `sp_read_port_*` and `sp_write_port_*` reservations as cross-unit shared resources.
- `tests/test_memory_contention.py` now validates:
  - DRAM contention
  - same-bank scratchpad contention
  - different-bank scratchpad overlap
  - scratchpad write-port contention
  - scratchpad read-port contention
- Full verification is green with `uv run pytest tests -q` at 45 passing tests.
