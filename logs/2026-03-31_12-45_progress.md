# Progress Log

2026-03-31 12:45:51 PDT

## Current Progress

- Upgraded shared-resource tracking from simple “busy until” timestamps to true cycle-interval reservations.
- Refined DMA shared-resource pressure so scratchpad banks, scratchpad ports, and DRAM are reserved only during the transfer phase after DMA setup.
- Added regressions that prove a short same-bank load may slip through DMA setup while active-phase conflicts still stall correctly.

## Current Implementation Status

- `perf_modeling/timing/resources.py` now supports interval-overlap checks between reservations.
- `perf_modeling/timing/scoreboard.py` now stores per-resource reservation intervals and checks overlap against proposed issue windows.
- `perf_modeling/engine.py` now validates and reserves shared resources using each reservation’s full `[start_cycle, end_cycle)` interval.
- `perf_modeling/isa/semantics.py` now delays DMA memory and scratchpad reservations until `setup_cycles` have elapsed.
- `tests/test_memory_contention.py` now covers:
  - active-phase same-bank contention
  - active-phase port contention
  - different-bank overlap during active DMA
  - setup-window same-bank load overlap before DMA transfer pressure starts
- Full verification is green with `uv run pytest tests -q` at 46 passing tests.
