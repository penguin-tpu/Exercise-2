# Progress Log

## Timestamp

2026-03-31 11:39:37 PDT

## Current Progress

- Extended the scalar RV32I core with a minimal machine-mode CSR file and structured machine-trap delivery.
- Added decode and execution support for CSR instructions and `mret`.
- Reworked fault paths so illegal instructions and misaligned accesses can trap architecturally instead of only halting via generic Python exceptions.
- Added end-to-end tests for CSR reads and writes, CSR dependency scoreboarding, trap-vector execution through `mtvec`, `mepc`/`mcause`/`mtval` population, `mret` return, and scratchpad-window load/store behavior.

## Current Implementation Status

- The simulator now models these additional machine-mode features:
  - CSR storage for `mstatus`, `misa`, `mtvec`, `mscratch`, `mepc`, `mcause`, `mtval`, `mhartid`
  - counter CSR reads for `cycle`, `instret`, `mcycle`, `minstret` and high halves
  - CSR instructions `csrrw`, `csrrs`, `csrrc`, `csrrwi`, `csrrsi`, `csrrci`
  - trap return via `mret`
  - structured illegal-instruction placeholders during decode so an unsupported word traps when executed instead of failing binary load immediately
- Trap handling is configurable:
  - default behavior still halts on traps for the simple bare-metal flow
  - optional trap-vector delivery through `mtvec` is available when `enable_trap_handlers=True`
- Verification is currently green:
  - `python3 -m compileall perf_modeling tests`
  - `uv run python -m unittest tests.test_rv32i tests.test_rv32i_engine`
  - `uv run python main.py`

## Current Roadblocks

- The environment still emits a Torch warning because NumPy is not installed in the `uv` environment.
- Network escalation is unavailable in the current tool policy, so any required `git push` may be blocked even after a local milestone commit.
- The simulator is still scalar-first; vector/tensor execution paths remain scaffold-level.

## Current Questions

- Whether the next execution milestone should prioritize privileged-machine completeness beyond the current CSR subset, or pivot to the accelerator-side vector/tensor instructions from the original scaffold plan.
- Whether to keep scratchpad as a fixed memory aperture or promote it to a more explicit memory-map object before DMA/vector activity is added.

## Pending Design Decisions

- How much privileged RISC-V behavior to model before returning focus to accelerator-specific instructions.
- Whether to add CSR side-effect and privilege checking beyond the current machine-mode-only subset.
- Whether to model instruction-access faults for unmapped PCs explicitly, or keep the current “fall off the end” completion rule for bare-metal programs without handlers.

## Next-Step Plan

- Build the first real accelerator-side vector/tensor placeholder slice on top of the now-stable scalar and trap machinery.
- Add mixed DRAM/scratchpad microbenchmarks that exercise more of the memory hierarchy timing path.
- Expand ELF and machine-level tests with more function-call and handler-style binaries.
