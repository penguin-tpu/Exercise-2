# Progress Log

## Current Progress

- Split frontend fetch-stall accounting by control-flow cause instead of keeping only one aggregate counter.
- Added a dedicated CLI `fetch` report and extended the shared summary/manifest output with fetch-stall breakdown data.
- Expanded RV32I and CLI regressions to pin the new counters and report formatting.

## Current Implementation Status

- `perf_modeling/state/arch_state.py` now carries `fetch_stall_reason` so the engine can preserve stall cause while the frontend is blocked.
- `perf_modeling/engine.py` now tags control instructions into stable fetch-stall buckets and records both `fetch_stall_cycles` and per-reason counters such as `fetch_stall.branch_cycles`.
- `perf_modeling/isa/semantics.py` clears the fetch-stall reason when control instructions resolve and the frontend becomes eligible to fetch again.
- `scripts/run_sim.py` now includes fetch-stall summary data in the compact run summary and supports `--report fetch`.
- `tests/test_rv32i.py` and `tests/test_run_sim_cli.py` verify the new counters, summary data, manifest schema, and CLI output.
