## Current Implementation Status

- Added `perf_modeling/perfetto.py` to build Perfetto/Chrome-trace JSON payloads from retained simulator trace records.
- The Perfetto export includes:
  - metadata tracks for frontend, scalar, load/store, DMA, vector, MXU, system, and summary lanes
  - paired issue/complete spans when a retained instruction trace can be matched
  - instant stall and trap markers
  - summary counter events for cycles, issued instructions, retired instructions, fetch-stall cycles, and peak event-queue depth
- `scripts/run_sim.py` now accepts `--perfetto-trace` and writes the generated JSON either to a file or stdout.
- `tests/test_run_sim_cli.py` now covers the new export path.
- `README.md` now documents the Perfetto export option.
