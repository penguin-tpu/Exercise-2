## Current Implementation Status

- Multiple execution paths intentionally still raise `NotImplementedError`, including:
  - decode
  - instruction planning
  - engine issue path
  - workload generation
  - packed quantization helpers

## Current Roadblocks

- There is no first working vertical slice yet, so the engine cannot execute a real program.
- The instruction planning interface exists, but there is no semantics registry wiring into the engine.
- The current unit models are occupancy stubs and do not yet enforce real scheduling or completion behavior.

## Current Questions

- Whether the first executable slice should start with a minimal generic instruction set or with accelerator-specific opcodes matching the eventual ISA.
- Whether tensor register writes should store full tensors directly or store more descriptor-heavy handles once memory movement semantics are implemented.

## Pending Design Decisions

- Exact first instruction set for the initial executable path:
  - load tile
  - store tile
  - mxu matmul
  - vector elementwise op
  - fence or wait
- How much scratchpad banking realism to include in the first working model.
- Whether early microbenchmarks should be built through `ProgramBuilder` only or also through a future textual decode path.

## Next-Step Plan

- Wire `Instruction.plan()` and the semantics registry into `SimulatorEngine.try_issue_head_instruction()`.
